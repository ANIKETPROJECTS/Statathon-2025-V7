## WHAT IS BROKEN (Root Cause Analysis)

The Rank Swapping output shows **Compliance Status: FAIL** even though the algorithm
ran correctly. There are **3 bugs** causing this:

---

### BUG 1 — `Land_Cultivated` silently skipped (CRITICAL)

**What happened:**
The user selected 3 target columns: `HH_Size`, `Land_Cultivated`, `No_NREG_Card`.
The report says `Columns Swapped: 2` and `Target Columns: HH_Size, No_NREG_Card`.
`Land_Cultivated` was never processed.

**Why:**
`Land_Cultivated` has dtype `str` — its values look like `"21.5 acres"`, `"1.7 acres"` etc.
The rank swapping code does `if col is not numeric: continue` and silently skips it,
without telling the user.

**The spec says** (from `SDC_Privacy_Enhancement_Module_Spec.md`, Section 4.2):
```python
for col in target_cols:
    if col is not numeric:
        continue
```
This skip is valid — but the UI must **warn the user before Apply** and must **not count
skipped columns in the "Target Columns" parameter display**.

**Fix required:**

1. **Pre-Apply validation warning** — before the user hits Apply Technique, show in the
   Pre-Flight Check panel:
   ```
   ⚠️  Land_Cultivated — skipped (string type: "21.5 acres")
        Rank Swapping requires pure numeric columns.
        Suggestion: Strip " acres" suffix and convert to float, or deselect this column.
   ```

2. **Post-Apply report** — in the report, under "Target Columns", only list columns that
   were actually processed. Do NOT list skipped columns. Show separately:
   ```
   Target Columns (processed) : HH_Size, No_NREG_Card
   Skipped Columns             : Land_Cultivated (non-numeric — string values)
   ```

3. **UI column list filter** — in the Target Columns checklist (left panel), for Rank
   Swapping only show columns where `dtype in [int64, float64]` AND `nunique > 10`.
   `Land_Cultivated` should NOT appear as a selectable option at all because its dtype
   is `str` (object). The auto-detection logic in `PRIVACY_ENHANCEMENT_CONFIG_SPEC.md`
   Section A0 already specifies this:
   ```
   Target Columns (numeric techniques): show only dtype in [int64, float64] with n_unique > 10
   ```
   The UI is currently showing `Land_Cultivated` (dtype=str) as a numeric target option.
   **This is the UI rendering bug.**

### BUG 2 — Compliance FAIL triggered incorrectly

**What happened:**
The compliance badge shows `FAIL` (red). But looking at the actual metrics:
- Marginal Preservation: CONFIRMED ✅
- Spearman ρ = 0.9446 (well above 0.85 threshold)
- Both processed columns have perfect stats

**Why it fails:**
The compliance check is comparing `len(selected_target_cols) == len(processed_cols)`.
User selected 3, only 2 were processed → mismatch → FAIL.

This is a **false failure**. The compliance should PASS because the algorithm succeeded
on all columns it could process. The skipped column is an input data issue, not an
algorithm failure.

**Fix required:**

Change the compliance check logic from:
```python
# WRONG
compliance_pass = (len(selected_cols) == len(processed_cols))
```
To:
```python
# CORRECT — from SDC spec Section 4.4
compliance_pass = (
    avg_spearman_rho >= 0.85               # utility threshold
    and marginal_preservation == True       # distribution preserved
    and swap_rate > 0                       # at least some swapping occurred
)
```

If any columns were skipped, show a **separate yellow warning** (not a red FAIL):
```
⚠️  WARNING: 1 selected column was skipped (Land_Cultivated — non-numeric).
    Algorithm ran successfully on 2 of 3 selected columns.
```

The FAIL badge should only appear if:
- Spearman ρ < 0.70 (severe distortion)
- Marginal preservation is violated (impossible if algorithm is correct)
- Zero columns were actually processed

### Change 2 — Pre-flight check (before Apply button)

```python
def build_preflight_rank_swapping(df, selected_target_cols):
    checks = []
    processable = []
    skipped = []

    for col in selected_target_cols:
        if df[col].dtype in ['int64', 'float64']:
            processable.append(col)
        else:
            skipped.append({
                'col': col,
                'reason': f'dtype={df[col].dtype}, sample value: "{df[col].iloc[0]}"'
            })

    if processable:
        checks.append({
            'label': f'Target columns (processable): {len(processable)}',
            'status': 'pass',
            'message': ', '.join(processable)
        })

    for s in skipped:
        checks.append({
            'label': f'{s["col"]} — will be SKIPPED',
            'status': 'warn',
            'message': f'Non-numeric: {s["reason"]}. Deselect or convert to numeric first.'
        })

    if not processable:
        checks.append({
            'label': 'No processable columns',
            'status': 'fail',
            'message': 'All selected columns are non-numeric. Select numeric columns only.'
        })

    return checks
```

### Change 3 — Compliance evaluation logic

```python
def evaluate_rank_swap_compliance(processed_cols, selected_cols, col_stats, skipped_cols):
    """
    Returns compliance status and reason.
    per SDC spec Section 4.4 metrics.
    """
    if not processed_cols:
        return 'FAIL', 'No columns were processed.'

    avg_rho = sum(s['spearman_rho'] for s in col_stats.values()) / len(col_stats)
    marginal_ok = all(s['marginal_preserved'] for s in col_stats.values())
    swap_occurred = any(s['swap_rate'] > 0 for s in col_stats.values())

    if not marginal_ok:
        return 'FAIL', 'Marginal preservation violated — implementation error.'
    if avg_rho < 0.70:
        return 'FAIL', f'Avg Spearman ρ = {avg_rho:.4f} < 0.70 threshold — too much distortion.'
    if not swap_occurred:
        return 'FAIL', 'No swaps occurred — increase swap fraction or check dataset size.'

    # Algorithm succeeded. Skipped cols = warning, not failure.
    if skipped_cols:
        return 'PASS_WITH_WARNING', (
            f'Algorithm succeeded on {len(processed_cols)} columns. '
            f'{len(skipped_cols)} column(s) skipped (non-numeric): {", ".join(skipped_cols)}'
        )

    return 'PASS', f'All {len(processed_cols)} columns processed successfully.'
```

Then in the report HTML:
```python
if status == 'PASS':
    badge_color = 'green'
    badge_text = 'PASS'
elif status == 'PASS_WITH_WARNING':
    badge_color = 'orange'   # yellow/amber — NOT red
    badge_text = 'PASS ⚠️'
else:
    badge_color = 'red'
    badge_text = 'FAIL'
```
